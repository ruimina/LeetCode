Implement wildcard pattern matching with support for '?' and '*'.
实现通配符匹配？和*

自己的思路是同之前的正则匹配.和*，但是超时了
用递归*考虑匹配1个或不匹配

参考：用两个指针从两个字符串开始循环，直到待匹配字符串
循环结束。如果一样或匹配的为？则两者++；若匹配的为*
则记录当前*号为位置和*号对应的待匹配字符，当做不匹配。
若匹配失败，但之前有*号，则退回星号的位置，让星号多匹配
一个字符。
待匹配字符匹配结束后，通过匹配字符串是否全是*判断是否匹配成功。

说明：每次匹配失败，只需要返回到上一个匹配的*号，之前的*号
都可以不考虑了，这是因为返回上个*号匹配失败，而返回上上个
*号匹配成功，则必然可以让上个星号多匹配一段来匹配成功，多匹配
的片段为上上个星号上次匹配的位置到上上个星号匹配成功时匹配的位置。
